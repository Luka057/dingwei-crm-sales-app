# 技术栈选型

## 前端

- React：适合复杂交互和组件化开发。
- TypeScript：减少业务字段和接口对接错误。
- Vite：开发启动快，适合中小型业务系统。
- React Router：四个主导航和详情页路由。
- TanStack Query：接口缓存、加载状态、错误状态。
- Zustand：轻量全局状态，如登录用户、筛选条件。
- CSS Modules 或 Tailwind CSS：快速构建移动端样式。
- PWA：内网手机访问时可添加到桌面。

## 后端

- Node.js + NestJS：结构清晰，适合接口、权限、AI 工具调用。
- Prisma：类型安全的数据访问。
- PostgreSQL：客户、拜访、计划、周报等关系数据。
- Redis：缓存、验证码、任务队列。
- JWT：登录态和接口鉴权。

## AI

- DeepSeek API：对话和 Function Calling。
- 后端统一封装 AI 工具，避免前端直接访问密钥。
- AI 工具函数包括查询客户、查询拜访、查询样板、生成周报、找板。

## 部署

- Docker Compose
- Nginx
- PostgreSQL
- Redis
- 后端服务
- 前端静态资源

## 选择理由

该技术栈学习成本适中，生态成熟，适合内网部署和后续扩展老板端、主管端。
