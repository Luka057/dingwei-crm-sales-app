# 业务需求说明

## 1. 背景

鼎伟织带工厂当前客户管理存在客户归属不清、销售工作难验证、拜访内容无沉淀、客户照顾情况不透明、订单数据分散、打样转化不可见等问题。

业务人员端负责从源头沉淀客户和拜访数据，为老板端和主管端提供真实、可追溯的数据基础。

## 2. 业务场景

### 2.1 每天上班

业务员打开日历，查看今天要拜访哪些客户、有哪些电话或微信跟进、是否有自定义事项。

### 2.2 拜访前

业务员进入客户详情，查看联系人、地址、上次拜访内容、样板和订单情况。

### 2.3 拜访后

业务员填写拜访内容，AI 自动生成摘要，沉淀到客户时间线。

### 2.4 每周结束

业务员进入周报页，AI 根据本周拜访计划和拜访摘要生成周报草稿，业务员补充备注和附件后提交。

### 2.5 找板

业务员拿到客户图片或工艺要求后，进入 AI 找板，输入图片、宽度、开力度、织带类型、颜色要求，匹配历史板样。

## 3. 权限规则

- 销售只能查看自己负责的客户、计划、拜访、周报。
- 主管可以查看分管销售的数据。
- 老板端通过接口查看全量数据。

## 4. 客户等级规则

- A 类客户：30 天未拜访视为超期。
- B 类客户：40 天未拜访视为超期。
- C 类客户：无强制超期要求。

## 5. 拜访方式

- 线下
- 电话
- 微信

## 6. 客户意向

- 进展良好
- 下单有望
- 持平等待
- 没有意向

## 7. 样板状态

- 起样
- 中板
- 确认
- 已转订单
- 已放弃
